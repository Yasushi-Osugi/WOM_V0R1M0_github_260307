STEP 2 — Understand the WOM Planning Model
------------------------------------------------------------

The WOM system is a planning kernel based on the following core entities.

Lot
FlowEvent
DemandEvent
StateView
TrustEvent
Operator
Resolver
PlanningKernel

Core architectural rules:

1. Flow/Event is the source of truth
2. State is a derived view
3. Operators modify planning objects (lots / flows)
4. StateView must never be directly mutated
5. Planning must remain deterministic and explainable

Planning loop:

DemandEvent generation
↓
Flow simulation
↓
State derivation
↓
Trust event detection
↓
Evaluation
↓
Resolver decision
↓
Operator application
↓
Re-simulation
